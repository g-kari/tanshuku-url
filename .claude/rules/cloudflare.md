# Cloudflare Workers ルール

## バインディングアクセス

Workerのバインディング (DB, URL_CACHE, ASSETS等) は必ず `c.env.〇〇` 経由でアクセスする。
グローバル変数やモジュールレベルの初期化は使わない。

## 非同期処理

レスポンス返却後の副作用処理 (クリック記録など) は必ず `c.executionCtx.waitUntil()` を使う。
`await` のまま放置するとリクエスト終了後にプロセスが打ち切られる。

## D1クエリ

- 必ず `.prepare()` → `.bind()` → `.run()` / `.first()` / `.all()` の形式
- SQLインジェクション防止のため文字列結合でのクエリ構築は禁止

## KVキャッシュ

- TTL は必ず `expirationTtl` (秒数) で指定
- キーの命名: `url:{code}` 形式を維持

## エラーハンドリング

- 404: `c.json({ error: '...' }, 404)`
- 410: 有効期限切れ
- 451: 安全でないURL
